Problem Two: What is the relationship between the diagnosis results and sleep?

题目附带的表格II中共有6000多个样本，每个样本包含年龄、性别、性质、病种、睡眠质量、入睡时间、睡眠时间、睡眠效率、精神质指数、日间行为障碍指数。每一项特征的评分都为0，1，2，3中的一个，按照原题解释是：The relevant scores for sleep conditions are given in Annex II (“0” for good, “1”
for normal, “2” for poor, and “3” for very poor), the higher the score, the worse the
sleep condition. 也就是说，评分越高说明睡眠条件越差，简单而言，就是越低对睡眠“质量”越有好处。(个人觉得这个评分机制不是很好)

解决流程

1.采取多个特征选择模型进行特征重要性的筛选 > 期望目的：能够选出重要性比较显著的特征再来分类，减少无关特征

2.给表中的9个特征进行打分评比，观察模型选择结果



From Table, we could directly see that the feature scores given via Lasso and Ridge are relatively close, which also indicates that most of the features are relatively independent. And when using the Recursive Feature Elimination (RFE), we found that the feature score of Sleep Latency and Sleep Time add up to one, which is quite conceivable in logic. And in this case, Sleep Latency is a relatively key factor.
From Fig.6, we could get the following conclusion:

  1. It is obvious that Sleep quality has the highest feature score, which indicates that Sleep quality has the biggest impact on health. If Sleep quality drop down, it would be of higher possibility for one to be diagnosed with all the listed diseases.
  2. We could drop the feature Hypnagogue, because it scores low in each method.

    得到我们一个想要实现的目标，去除了一个最弱的特征，无论在哪个特征选择模型中都得分最低的Hypnagogue，在后续分类过程中将去除。

3.我们在此处使用的特征选择模型包括：

A. Mutual Information and Maximal Information coefficient (MIC)

p(x,y) is the joint probability distributions function of X and Y; p(x) and p(y) is the marginal probability distribution function of X and Y.
By referring to the article of Kinney(2014), because both the dependent and independent variables are discrete, using I(X,Y) to analyze the correlation is easy and practical. And later, we would make a comparison of these two methods.
由于本题的自变量特征和因变量的取值都属于离散型，因此用I(X,Y)来评估相关性具有易用性和可行性。

B. L1 Regularization /LASSO

L1 regularization adds the L1 norm of the coefficient ω as a penalty to the loss function, which forces the coefficients corresponding to those weak features to become zero because the regular term is non-zero. Therefore, regularization of L1 tends to make the regression model sparse (coefficient ω is often 0). This feature makes L1 regularization a good feature selection method.
L1正则化由于加了一次惩罚项能够使微相关的特征系数加快降为0，在保留强特征的同时也可能会造成模型稀疏化，但我们就是用这个来找强特征，所以稀疏化在在某种意义上来说可以使我们更直观地找到强特征，下面还有它的加强版，大大增强了稀疏化无关系数的效果。

C. Its improved LASSO Model: Random LASSO

In order to select the strongest feature, we apply the stability theory here and the Random LASSO method. Stability selection is an advanced method based on the combination of subsampling and selection algorithms. The selection algorithm can be regression, SVM or other similar methods. By referring to Meinshausen(2010), we used Radom LASSO. Its main idea is to run feature selection algorithms on different subsets of data and subsets of features, continuously repeating and finally summarizing feature selection results, such as the ability to count the frequency at which a feature is considered as an important feature (the number of times of being selected as an important feature divided by the number of times the subset it is tested on).
Ideally, important features score close to 100%. A slightly weaker feature score will be a non-zero number, while the most unwanted feature score will be close to zero.
稳定性选择就是将二次抽样和选择算法结合起来，将稳定性选择的思想应用到Lasso身上，可以最终得到最稳定的特征，同时把不是很稳定的特征去除，虽然去除的特征可能是比较重要的，但是我们能够断定保留下来的特征是足够强的，至于更均匀的特征保留我们交给其它模型来找。

D. L2 Regularization/ Ridge Regression

L2 regularization adds the L2 norm of the coefficient vector to the loss function. Since the coefficients in the L2 penalty term are quadratic, there are many differences between L2 and L1. The most obvious point is that L2 regularization averages the coefficients. Liu (2007) pointed out that, for the associated features, they can get closer coefficients.
L2正则化由于加了一个二次惩罚项，所以它不会像L1正则化那样使模型变得稀疏，而是尽量地让模型各个特征的系数趋向均匀，这个可以让我们找到相关性(耦合性)比较强的特征，弥补了上面稳定性选择方法的缺陷。

E. Its improved Model: RFE Model

By referring to Granitto (2006), we proposed an improved Ridge Regression Model: RFE Model. The main idea of recursive feature elimination is to build a model iteratively (such as SVM or regression model) and then select the best (or worst) features (which can be selected based on the coefficients). And we put the selected features aside and then repeat this process for the remaining features until all the features have traversed. The order in which the features are eliminated in this process is the order of the features. Therefore, this is a greedy algorithm for finding the best subset of features.
The stability of the RFE depends on which model the underlying layer uses in the iteration. For example, RFE is not stable if the normalized regressions used by RFEs are not stable, and RFE is unstable if Ridge regressions are stable and Ridge regularized regressions are stable.
递归特征消除的主要思想是反复的构建模型（如SVM或者回归模型）然后选出最好的（或者最差的）的特征（可以根据系数来选），把选出来的特征放到一遍，然后在剩余的特征上重复这个过程，直到所有特征都遍历了。这个过程中特征被消除的次序就是特征的排序。因此，这是一种寻找最优特征子集的贪心算法。
RFE的稳定性很大程度上取决于在迭代的时候底层用哪种模型。例如，假如RFE采用的普通的回归，没有经过正则化的回归是不稳定的，那么RFE就是不稳定的；假如采用的是Ridge，而用Ridge正则化的回归是稳定的，那么RFE就是稳定的。

对于同一种病，睡眠质量越差(3是最差的)的人群中患的人数更多。

稍微总结一下

我们成功在特征选择过程中找到两个弱特征并去除，现在还剩下7个特征在后续分类算法中要用于训练和评估。
By analyzing the scatter point and the feature selection model results, we could drop two features Sex and Hypnagogue. The reason why we don’t take Sex into account is that the spread of diseases in male and female are almost the same which means that Sex is a partial unrelated factor. The reason why we could delete Hypnagogue is because it scores the lowest in each model.

Problem Three: Assuming you are a doctor, what diagnosis would you make to the patient based on the data in Annex III?

实际上就是一个分类预测问题

按照一位数据分析程序猿标准的思维，就顺着“如果能够在有限且可接受的时间范围内遍历所有可能性得到最优结果，那么就应该往这个方向继续分析和优化”的思路，我把剩余的7个特征进行了组合，准备结合多个分类模型进行暴力寻优一波，然后惊喜出现了。

采用的分类模型有6个：(在图中以左侧命名)
MultNB: Multinomial NB
GausNB: Gaussian NB
BernNB: Bernoulli NB
KNN: Neighbors Classifier
RF: Random Forest Classifier
DT: Decision Tree Classifier
Based on the results from problem 2, we drop two features, which are the Sex and Hypnagogue.
特征组合有99种，我是准备以3个特征起步，简单点说，就是3个特征、4个特征、5个特征、6个特征、7个特征的组合(特征不分顺序)，一共有99种
Thus here we used 99 combinations of seven features for classification training and prediction (Set the least selected number of features as 3). The training set and the prediction are the same here. In the following analysis, we would start from three features for each classifier. And if by adding one feature to the classifier, the prediction accuracy of the classifier would improve, then we would add it to the classifier.
分类要用到的特征有7个：(分别用0-6表示)
Seven features is represented using 0-6, which are: {‘Age’, ‘Sleep quality’, ‘Sleep latency’, ‘Sleep time’, ‘Sleep efficiency’, ‘Sleep disorder’,’Daytime dyfunction’}

样本有限，选择在整个训练集上测试

第一个看不出什么结果
The x axis is the type of classification method and the y axis is the prediction accuracy rate. Here we could see that when there are three features, the classification method with the highest accuracy rate is DT, which is Decision Tree Classifier.

第二个开始有点眉目了，
we could see that [DT]0.502(0/1/2/4) > [DT] 0.493(0/1/2/3). Thus, it means that the importance of feature 4 > 3, which means that Sleep efficiency > Sleep time.
得到两个有重要排序的特征
(这里采用的就是高中单一实验变量的实验思想)

开始出现三个特征重要性的排序了
we could get [DT]0.515(0/1/2/6) < [DT]0.517(0/1/3/6) < [DT]0.526(0/1/4/6) . Thus, Feature 4 > 3 > 2, which means that the feature importance sequence is: Sleep efficiency > Sleep time > Sleep latency

再次验证了前两次排序的结果！
we get that: [DT]0.639(0/1/2/3/6) < [DT]0.645(0/1/2/4/6), thus: Feature 4 >3, which means Sleep efficiency > Sleep time Also, we could get: [DT]0.652(0/1/3/4/6) > [DT]0.645(0/1/2/4/6); Thus, feature 3 > 2, which means Sleep time > Sleep latency

建模总结：本来只是想简单地输出准确率和对应特征组合，没想到引出了一个用于评估特征重要性的巧妙方法，就是利用同一分类方法在不同特征组合(只有一个特征变量)情况下的分类准确率来评估特征重要性，若使用了这个特征能够分类更加准确，那么有理由说明这个特征比之前那个特征更重要，这也跟特征选择模型RFE思想有点类似。不需要进行拓扑排序，第二个结果直接验证特征排序的准确性，这更进一步体现了排序的可信度，这也从侧面反映了我们在题二中去除的特征的确是比较无关的，才能够实现这样的分类效果，才能够通过剩余的有一定关联的特征里发现重要性排序。
